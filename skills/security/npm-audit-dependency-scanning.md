# 의존성 보안 스캐닝 (SCA)

**ID:** `SKL-DEPENDENCY-SCANNING`  
**범위(Scope):** AI Recommended  
**우선순위:** High  
**적용 조건:** 소프트웨어 구성 분석(SCA)으로 서드파티 의존성의 알려진 취약점을 탐지·추적·패치하는 범용(foundational) 자동화 표준. CI 게이팅, 자동 패치 PR, 심각도 기반 우선순위, 잠금 파일·의존성 분리를 다룬다. CI에 취약점 스캐닝을 통합하거나, 의존성 자동 업데이트를 설정하거나, 새 라이브러리를 채택할 때 읽는다.

---

## 지시사항 (Instructions)

1. 자동으로·지속적으로 탐지한다: 취약점 DB는 매일 갱신되므로 1회성 점검으로 끝나지 않는다. CI와 정기 스케줄로 반복 실행해 새로 공개된 취약점도 잡는다.
2. 심각하면 막고, 패치는 받아 온다: 일정 심각도 이상은 빌드에서 게이팅하고(경고만 하면 아무도 안 본다), 보안 패치는 자동화 도구가 PR로 올리게 해 사람은 검토·머지에 집중한다.
3. 공격 표면을 작게·재현 가능하게: 잠금 파일을 커밋해 어디서든 같은 의존성 트리가 나오게 하고, 프로덕션엔 런타임 의존성만 넣으며, 새 라이브러리는 들이기 전에 검증한다.
