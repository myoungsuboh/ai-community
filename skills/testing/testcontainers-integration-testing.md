# 통합 테스트: DB·API·컴포넌트 연동

**ID:** `SKL-INTEGRATION-TESTING`  
**범위(Scope):** AI Recommended  
**우선순위:** High  
**적용 조건:** 실제 DB·외부 서비스·API를 포함한 컴포넌트 간 협력을 검증하는 범용 통합 테스트 표준: 테스트 전용 의존성, 테스트 간 격리(초기화·정리), 계약(스키마·상태·헤더) 검증, 외부 서비스 샌드박스화, CI 재현성. API 계약·DB 연동·외부 서비스 연동을 테스트하거나 CI에서 의존 서비스를 띄워 검증할 때 읽는다. 특정 언어/프레임워크/도구에 무관하다.

## 지시사항 (Instructions)

1. 프로덕션을 건드리지 않는다: 테스트 전용 DB·큐·캐시·버킷만 쓰고, 실제 프로덕션 데이터에 연결하지 않는다.
2. 테스트 간 격리: 각 테스트는 알려진 상태에서 시작하고 끝나면 흔적을 지운다(롤백/정리): 실행 순서와 무관하게 같은 결과가 나와야 한다.
3. 구현이 아니라 계약을 본다: 진짜 요청을 보내 외부에서 관찰 가능한 계약(스키마·상태/오류 코드·헤더·페이로드)을 검증한다.
4. 외부 부수효과는 대역으로: 결제·메일·SMS 등은 샌드박스/가짜 서버로 대체해 진짜 과금·발송·외부 변경을 일으키지 않는다.
5. 결정적이고 재현 가능하게: 시간·랜덤·타이밍·순서에 의존하지 않고(비동기는 완료를 기다린 뒤 단언), 의존 서비스를 코드로 띄워 로컬과 CI가 같게 한다.
